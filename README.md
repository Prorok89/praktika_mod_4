# Image Processor

Система динамической обработки изображений на языке Rust с поддержкой плагинов в виде динамических библиотек (`cdylib`).

## Возможности

Программа позволяет применять различные эффекты к PNG-изображениям, загружая соответствующий плагин во время выполнения.

### Доступные плагины

- **Mirror**: Зеркальное отображение изображения по горизонтали и/или вертикали.
- **Blur**: Эффект размытия (box blur) с настраиваемым радиусом и количеством итераций.

## Структура проекта

- `image_processor`: Основное приложение, отвечающее за загрузку изображений, работу с файлами и динамическую загрузку плагинов.
- `mirror_plugin`: Плагин для зеркалирования.
- `blur_plugin`: Плагин для размытия.

## Сборка и запуск

### Сборка проекта

Для сборки всех компонентов используйте команду:

```bash
cargo build
```

### Использование

Запуск основного приложения осуществляется через CLI:

```bash
cargo run -p image_processor -- \
    --input <путь_к_файлу.png> \
    --output <путь_сохранения.png> \
    --plugin <mirror|blur> \
    --params <путь_к_файлу_параметров.txt> \
    --plugin-path <путь_к_директории_с_dll>
```

### Пример параметров

Для плагина **Mirror** (`mirror.txt`):

```json
{
  "horizontal": true,
  "vertical": false
}
```

Для плагина **Blur** (`blur.txt`):

```json
{
  "radius": 2,
  "iterations": 3
}
```

## Тестирование

В проекте реализованы модульные тесты для каждого компонента:

- Проверка корректности PNG-заголовков.
- Тестирование алгоритмов обработки (зеркалирование, размытие).
- Проверка граничных условий (нулевые размеры, некорректные параметры).

Запустить все тесты:

```bash
cargo test
```

## Архитектура плагинов

Плагины представляют собой динамические библиотеки, экспортирующие функцию `process_image` с C-совместимым интерфейсом (`extern "C"`). Это позволяет основному приложению загружать любые совместимые библиотеки без перекомпиляции ядра системы.
